### 3.3 Internal Linking Strategy

Every page should link to:
- At least 2 related service pages
- At least 1 industry page
- At least 1 blog post
- The pricing page or a CTA
- The contact page

Build topic clusters:
- **Website Design cluster**: service page → industry pages → blog posts about website ROI
- **Local SEO cluster**: service page → GBP service page → blog posts about local ranking
- **Pricing cluster**: pricing page → comparison pages → "how much does a website cost" blog post

### 3.4 Image SEO

- All images need descriptive `alt` text (not "image1.jpg" but "plumber-website-design-example-off-piste-studio")
- Use WebP format with AVIF fallback for modern browsers
- Lazy load below-the-fold images
- Include project screenshots on industry pages with industry-relevant alt text
- Add an image sitemap

### 3.5 Page Speed & Core Web Vitals

- Target LCP < 2.5s, FID < 100ms, CLS < 0.1
- Preload critical fonts (Bebas Neue, Inter)
- Inline critical CSS
- Defer non-critical JS
- Compress all assets
- Use CDN for static assets

- Add image sitemap for project screenshots
- Submit to Google Search Console and Bing Webmaster Tools

### 6.2 URL Structure

All URLs should be:
- Lowercase, hyphenated
- Descriptive and keyword-containing
- Shallow (max 3 levels deep): `/industries/plumbers`, not `/services/industries/trades/plumbers`
- No trailing slashes (pick one convention and stick to it)
- No file extensions in production (use clean URLs, not `.html`)

### 6.3 Redirects

When restructuring:
- 301 redirect any changed URLs
- Map old sitemap URLs to new structure
- Monitor 404s in Search Console weekly for first 2 months

### 6.4 Mobile Optimization

- The site is already responsive — verify all new pages pass mobile-friendly test
- Ensure tap targets are at least 48px
- Click-to-call links on all mobile views
- Test on real devices, not just Chrome DevTools

### 6.5 Security

- Ensure HTTPS on all pages
- Add security headers (X-Frame-Options, CSP, etc.)
- Keep dependencies updated

### 8.2 llms-full.txt

`/llms-full.txt` should be a comprehensive, structured document containing:

### 8.3 Structured Content for AI Retrieval

To be surfaced in LLM answers (ChatGPT, Perplexity, Gemini, etc.):

- **Write in Q&A format** in FAQ sections — LLMs love extracting these
- **Use clear, factual statements** — "Off Piste Studio builds websites for plumbers starting from $X" not "We craft bespoke digital experiences"
- **Include specific numbers** — pricing, timelines, number of clients, review ratings
- **Use schema markup** — FAQ, HowTo, Service schemas help AI understand content structure
- **Create definitive answers** to questions your ICP asks:
  - "How much does a plumber's website cost?"
  - "What should a dentist's website include?"
  - "Do tradies need a website?"
  - "Best web design agency in Perth for small business"
- **Maintain freshness** — AI models weight recent, updated content

## 12. COMPETITIVE EDGE

### What makes this strategy different:

1. **Industry pages are the moat.** Most agency websites have a generic "services" page. Having 20 dedicated industry pages means you rank for 20x the keywords with 20x the specificity. A plumber searching "plumber website design" lands on a page that speaks directly to them — not a generic agency portfolio.
